Feature Card

The feature card is meant to give users a preview of another page's content. Use feature cards to preview and direct users to the newest case study.

The feature card should be used only on the Home page. There should only be one feature card included on the page.

Use on:

  • Home page

Gallery Card

The gallery card acts as a large clickable button meant to give users a preview of a case study. Use gallery cards to preview and direct users to all case studies.

The gallery card should be used only on the Work page. Gallery cards should be displayed in a grid layout with a maximum of two cards per row.

Use on:

  • Work page

Project Card

The project card is availble to highlight a shortened list of information. Use project cards to give users a highlited list of important information like project details, process details, etc..

The project card should be used only within individual case studies. Multiple project cards can be used in a single case study. However, project cards should be not be placed next to each other.

Use on:

  • Case Study page

Accessibility

Ensure accessibility by using descriptive button labels for users using assistive technologies.

Also, inlcude descriptive alt text for all images used in cards.

Use

Use responsive buttons bar on all pages. To see specifics on placement, see page patterns

Best Practices

  • Ensure the buttons direct to a working existing page
  • Begin buttons with action oriented verbs Go to, Back to, or View the
  • Use specified background color White-4 (#F1F1F1) for Feature card background see design system color palette
  • Use Hover states with darkened shadows for Gallery cards
  • Use <ul> or <ol> elements for the lists on the Project cards
